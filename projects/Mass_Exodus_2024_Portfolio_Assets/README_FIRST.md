# Mass Exodus 2024 Portfolio Asset Library

This folder is organized to match the planned scrolling case-study experience.

## Recommended workflow

1. Add original assets to the most specific numbered folder.
2. Keep editable masters and final web exports separate.
3. Record every useful asset in `00_PROJECT_ADMIN/06_Asset_Tracking/asset_manifest.csv`.
4. Use clear filenames with the structure below.
5. Preserve proof of authorship and collaboration beside the related work.
6. Do not place private Slack messages, student data, contact details, or unapproved participant imagery on the public site.

## Naming convention

`ME24_[SECTION]_[ASSET]_[STATE]_[VIEW]_[NUMBER]_v01.ext`

Examples:

- `ME24_HERO_Runway_Final_Wide_01_v01.jpg`
- `ME24_LOGO_Stamped_Final_Black_01_v03.svg`
- `ME24_POSTER_General_Final_Front_02_v02.tif`
- `ME24_CREATIVEFEST_MuslinDrape_Process_Wide_01_v01.jpg`
- `ME24_BROADCAST_HoldingAnimation_Final_16x9_01_v01.mp4`

## Asset states

- `SOURCE` — untouched original
- `PROCESS` — development or behind-the-scenes material
- `PROOF` — evidence of use, authorship, scale, or impact
- `FINAL` — approved final output
- `WEB` — optimized public-site export

## Suggested web export targets

### Images
- Hero: 2400–3200 px wide, AVIF/WebP plus JPG fallback
- Full-screen case-study images: 2000–2400 px wide
- Supporting images: 1200–1800 px wide
- Thumbnails: 600–900 px wide

### Video
- MP4 H.264 fallback
- WebM when useful
- Muted autoplay loops should usually stay under 8–12 MB
- Create poster frames for every video
- Keep original masters outside the optimized folders

### Documents
- Preserve the original PDF or source document
- Export selected spreads as images for the webpage
- Avoid embedding large complete PDFs directly into the scrolling experience unless requested

## Folder logic

Folders `01` through `17` correspond directly to the webpage narrative.
`18_WEB_PRODUCTION` is for derivative files prepared specifically for development.
